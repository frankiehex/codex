# 開發執行計畫（AI 自助洗車 App）

## 目標與成果指標
- **正式上線時間**：12 個月內於雙平台（iOS/Android）推出正式版。
- **營運覆蓋率**：首年導入 50 個據點，次年擴展至 120 個據點。
- **使用者指標**：上線 3 個月內達到 2 萬月活躍用戶（MAU），付費轉換率 15%。
- **服務可用性**：核心 API 服務達到 99.5% SLA，洗車機控制指令成功率 99%。

## 里程碑時間表
| 時程 | 里程碑 | 主要產出 |
| --- | --- | --- |
| 第 0-1 月 | 團隊組建與需求定義 | 招募團隊、完成 PRD、定義系統需求、確立 API 合作意向 |
| 第 1-3 月 | MVP 設計與原型驗證 | 完成 UI/UX 原型、技術 PoC（洗車機、販賣機 API）、確立雲端架構 |
| 第 3-6 月 | MVP 開發與封測 | 建立後端微服務、行動 App MVP、管理後台、CI/CD 管線、站點排程資料導入 |
| 第 6-9 月 | 擴展整合與資安審查 | 導入支付、導航、優惠券系統，完成滲透測試、法遵與隱私審查，啟動封測 |
| 第 9-12 月 | 正式上線與優化 | 全台 20 個據點上線、蒐集使用者回饋、優化營運儀表板、規劃第二年功能 |

## 核心工作分解 (WBS)
### 1. 產品與設計
- 撰寫詳細產品需求文件（PRD）、使用案例、客戶旅程地圖。
- 製作 App Wireframe、High-fidelity Prototype，進行可用性測試。
- 設計品牌識別、插畫、App Store 行銷素材。

### 2. 後端服務
- 建立 API Gateway、Auth Service（OAuth 2.1 / OIDC），支援手機號碼註冊與簡訊 OTP 驗證流程。
- 實作 Location、Availability、Order/Payment、Integration Gateway 微服務。
- 建置訊息佇列與事件串流（Kafka / PubSub），進行機台遙測蒐集。
- 佈署於雲端（GCP/AWS）並使用 Terraform/IaC 管理。

### 3. 行動應用與前端
- Flutter/React Native App：地圖搜尋、即時排隊、優惠券錢包、支付與洗車控制。
- Next.js 管理後台：營運儀表板、站點管理、優惠券與商品管理。
- 整合 Firebase/OneSignal 推播、設定 Crashlytics 與分析事件。

### 4. 物聯與第三方整合
- 與洗車設備商協議 API/通訊協定，建立測試環境與模擬器。
- 建立中介閘道（MQTT/HTTPS）並實作命令簽章、回報機制。
- 與販賣機供應商整合商品資料與庫存同步。
- 導入支付（Stripe/Line Pay/信用卡 Tokenization）、導航（Google/Apple/HERE）、簡訊 OTP（Twilio、MessageBird 或在地業者）API。

### 5. 資安與合規
- 完成資料分類、資安政策、SOC 2/ISO 27001 自評。
- 進行滲透測試、程式碼安全掃描（SAST/DAST）。
- 設計事件回應流程、異常通報與備援演練。
- 建立隱私權政策、使用者條款、支付合規（PCI DSS）。

### 6. 營運與支援
- 建立即時客服（聊天/電話）與常見問題知識庫。
- 規劃據點維運流程、補貨、設備保修 SLA。
- 建立數據儀表板（MAU、轉換率、機台使用率）。
- 設計推廣活動：會員推薦、企業合作方案。

## 工具與流程
- **專案管理**：Jira/Linear 建立 Roadmap 與 Sprint 板，週會檢視 KPI。
- **版本控制**：GitHub Flow，PR 審查需通過單元測試、Lint、自動化部署。
- **CI/CD**：GitHub Actions or GitLab CI，部署至測試/生產環境採用 Canary。
- **品質保障**：自動化單元測試、API 測試（Postman/Newman）、E2E 測試（Detox/Appium）。
- **觀測性**：Prometheus + Grafana、OpenTelemetry、Sentry。

## 風險與緩解策略
| 風險 | 影響 | 緩解措施 |
| --- | --- | --- |
| 與設備商 API 整合延遲 | 延誤 MVP 開發時程 | 優先建立模擬器、簽訂 SLA、設定多家供應商備援 |
| 支付與資安審查時間過長 | 上線時程延後 | 及早提交審查、使用已通過認證的支付代收業者 |
| IoT 設備故障率高 | 客訴增加、營收損失 | 導入預防性維護、備援零件庫存、即時監控 |
| 使用者成長不如預期 | 收益不足 | 加強行銷活動、設計會員制與促銷策略、建立合作通路 |
